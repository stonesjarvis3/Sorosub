# Contributor Code of Conduct

## Our Pledge

We are committed to providing a welcoming and inspiring community for all. The SoroSubs project and its community are dedicated to providing a harassment-free experience for everyone, regardless of age, body size, disability, ethnicity, gender identity and expression, level of experience, nationality, personal appearance, race, religion, sexual identity and orientation, or socioeconomic status.

## Our Standards

Examples of behavior that contributes to creating a positive environment include:

- **Be Respectful**: Treat all community members with respect and dignity
- **Be Inclusive**: Welcome contributors of all backgrounds and experience levels
- **Be Collaborative**: Work together constructively and support fellow contributors
- **Be Constructive**: Provide helpful feedback and suggestions
- **Be Professional**: Maintain professionalism in all communications
- **Acknowledge Mistakes**: Admit when you're wrong and learn from feedback
- **Focus on Issues**: Discuss ideas and problems, not personalities

Examples of unacceptable behavior include:

- Harassment, intimidation, or discrimination of any kind
- Offensive comments related to protected characteristics
- Unwelcome sexual attention or advances
- Trolling, insulting/derogatory comments, and personal or political attacks
- Public or private harassment
- Publishing private information without consent
- Deliberate disruption of discussions or events
- Any other conduct that violates professional norms

## Scope

This Code of Conduct applies to all SoroSubs community spaces including:

- GitHub repositories and discussions
- Pull request and issue comments
- Code review feedback
- Community chat channels
- In-person and virtual events
- Email communications

## Enforcement

### Reporting Violations

If you experience or witness unacceptable behavior, please report it by:

1. **Direct Communication**: Contact the person if you feel safe doing so
2. **Maintainers**: Email conduct@sorosub.example with:
   - Description of the incident
   - Context and dates
   - Individuals involved (if comfortable sharing)
   - Any supporting documentation

3. **GitHub**: Use the "Report abuse" feature on GitHub

### Consequences

Violations of this Code of Conduct will result in:

1. **Warning**: First-time minor violations receive a private warning
2. **Temporary Ban**: Repeated violations result in temporary suspension
3. **Permanent Ban**: Serious or repeated violations result in permanent removal

The maintainers will:
- Investigate all reports fairly and impartially
- Respect privacy and confidentiality
- Respond within 7 days of a report
- Keep the reporter informed of actions taken

## Guidelines for Contributors

### When Opening Issues

- **Be Clear**: Provide specific, reproducible examples
- **Be Respectful**: Assume good intent from maintainers
- **Do Research**: Check if the issue has been reported before
- **Provide Context**: Include relevant error messages and system information

### When Submitting Pull Requests

- **Follow Style**: Adhere to project coding standards
- **Write Tests**: Include tests for new functionality
- **Document Changes**: Update relevant documentation
- **Review Your Own**: Self-review before requesting review
- **Be Responsive**: Address feedback promptly

### When Reviewing Code

- **Be Constructive**: Suggest improvements, not just criticism
- **Ask Questions**: Seek understanding before judging
- **Praise Good Work**: Acknowledge effort and good solutions
- **Focus on Code**: Review code, not the person
- **Be Timely**: Provide feedback within reasonable timeframes

### When Discussing Disagreements

- **Assume Good Intent**: Believe others are trying to help
- **Stay On Topic**: Focus on technical merits
- **Be Specific**: Reference code, documentation, or standards
- **Seek Consensus**: Try to find common ground
- **Escalate Respectfully**: Ask maintainers to mediate if needed
